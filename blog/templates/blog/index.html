Pandas DataFrame GUI

Data analysis is the main reason why I learnt Python. My working experience taught me that you can have lots of data, but knowing how to extract information is a different thing! This is a script in where a random Pandas DataFrame is created and plotted using matplotlib. A new set of values is created every second and the plot is updated automatically (after you press the button). The main goal in here was to plot the DataFrame graph inside the Tkinter GUI with the FigureCanvasTkAgg module from matplotlib.backends.backend_tkagg, as to have an executable program that allows you to work and generate plots, further than just showing a graph with the plt.show() function.

QRcode reader

Thanks to Python, you can generate a QR code in 4 lines of script using the qrtools package, while doing a QR code reader happens to be a bit more tricky. This reader is personally one of the most interesting projects I have done, given the versatility of elements to combine between mouse click events, image manipulation and using computer vision for image analysing.
The script starts by maximizing a transparent window on which the x and y coordinates of the mouse clicks will be used as variables with the mouse package to generate an image with the PIL ImageGrab package, triggering the actual GUI and showing the created image to scan, openCV is used to detect if the image contains a QR code and the data will be displayed. An 'image-file.png' will be created and you may need sudo privileges to run the app, also you will need an existing QR code to scan before executing the script. Once you run the app click the top-left corner followed by clicking the bottom-right corner of the code you want to read.

Shortest-path finder

Networks are objects that contain nodes and edges (between a pair of nodes) that can be used to represent a wide variety of real-world situations. A common problem where these appear is in finding the shortest route between two nodes in a network. For instance, this could be the shortest distance between two cities, where the nodes represent the cities and the edges are roads connecting them. In this case, the weights of the edges would be their lengths.

This code uses primarily the NetworkX package for working with trees and networks, aided by the Matplotlib package to plot the figure. The default algorithm for finding the shortest path between two nodes is Dijkstra's algorithm and can be used for problems as the traveling salesperson problem and the route inspection problem. The red color lines are the examined edges between nodes 7 and 9, and in contrast with blue color is the "shortest path", meaning "fewest edges". Weight is an optional keyword argument added to make some routes preferable to others.
